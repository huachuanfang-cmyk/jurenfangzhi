# 巨人纺织 ERP 系统

> 一套为**纺织 / 印染 / 服装面料**行业打造的轻量级 ERP，单文件 HTML + Supabase 云端，
> 双击即可运行，填一次公司信息就能用。免费开源，欢迎同行复用。

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](./LICENSE)

## ✨ 这套系统能做什么

覆盖纺织贸易/加工从**接单 → 投产 → 入库 → 出货 → 对账 → 收款**的完整闭环：

### 订单与加工
- **销售订单**：多颜色/多缸号、合同品名、按 KG 或按米计价、合同打印
- **加工跟踪**：织造/染色/印花/后整理全流程，加工费按工序记录
- **织厂加工单 / 染整加工单**：一键生成并打印委托加工单

### 原料与库存
- **纱线采购**：采购合同、到货登记、成本核算
- **胚布采购**：坯布订购合同
- **发料 / 回仓**：原料出库与余纱回仓
- **成品库存**：入库登记、出货管理、退货处理、库存台账

### 财务
- **应收对账**：自动匹配出货、按订单/跟单/布类拆分对账、收款账户管理
- **应收账龄分析**：0-30/30-60/60-90/90+ 天分档，催款优先级与坏账预警
- **客户毛利分析**：按客户/订单/月份三视图，工序成本拆分，成本漏录提醒
- **应付对账**：加工厂对账与付款确认
- **报价单**：客户报价管理与打印

### 系统与协作
- **公司档案**：公司名/地址/Logo/银行账户/制单人一处配置，全系统自动套用
- **分层级用户角色**：管理员/财务/跟单/采购/仓管/只读，菜单按角色隐藏
- **操作日志/审计**：谁、何时、改了什么单据，全程留痕
- **备份/恢复**：一键导出到 U 盘 + 历史快照回滚 + 多设备自动同步

## 🏗️ 技术架构

| 层 | 用什么 | 说明 |
|----|--------|------|
| 前端 | 纯 HTML + 原生 JavaScript（单文件 `index.html`） | 无框架、无构建，浏览器直接打开即可 |
| 本地存储 | localStorage | 离线可用，断网照常录入 |
| 云端同步 | [Supabase](https://supabase.com)（Postgres + Auth） | 多设备共享数据、账号登录 |
| 部署 | [Cloudflare Pages](https://pages.dev)（或任意静态托管） | 免费、全球 CDN |

**设计哲学：本地优先 + 云端同步。** 数据先存本地（永不丢），再实时上云（多设备共享）。
断网也能用，联网自动同步。

## 🚀 快速开始

详细步骤见 [DEPLOY.md](./DEPLOY.md)。三步概览：

1. **建云端数据库**：注册 Supabase，跑一遍 `supabase-schema.sql` 建表
2. **部署前端**：把 `index.html` 传到 Cloudflare Pages（或本地双击打开）
3. **填公司信息**：登录后进「公司档案」，填上你自己公司的名称/地址/Logo/银行账户 → 所有单据自动变成你的

## 🔒 数据安全

- **本地优先**：所有改动先写本地 localStorage，断网不丢数据
- **自动快照**：每张关键单据保存时自动留快照，误删可一键恢复
- **作废留痕**：业务单据禁止硬删除，只能作废（保留审计轨迹）
- **角色权限**：普通员工看不到不该看的菜单和财务数据
- **操作审计**：所有关键操作记录操作人，可追溯

## 🧪 质量护栏

本项目用一套**回归测试 + 行为锁定**机制保障稳定性：

```bash
node tools/guard/run-guard.mjs    # 运行全部护栏测试
```

- **回归测试**：核心计算、状态机、同步逻辑
- **行为锁定**（`locked-behaviors-tests.mjs`）：把每个已修复的 bug 锁住，未来任何改动若导致 bug 复发，测试立刻 FAIL 阻止合并
- **打印快照**：单据打印模板逐字对比，防止改一处坏一片

> 每次改代码前都应跑一遍 `run-guard.mjs`，全绿才提交。
